  • Files and File systems.
  • open() - reading, writing or both
  • unique descriptor, fd in kernel and shared with user space
  • Regular file - bytes of data, organized in linear array, called "byte stream"
  • ----before the zero pos------0123456.............-----after zero pos----
  • fd is zero then the file is opened
  • file position - file offset, C type in Linux, 64-bits
  • it is not possible to write before zero position, but is it possible to write after
  • truncation goes both ways! extension usually is putting in zeros
  • writing in a position "in the middle" overwrites the byte
  • the file size is called file length
  • internally kernel uses an information node (inode) as file handle (to find the file)
  • i-number/ino does NOT includes file name, but includes
    • - modification timestamp
    • - owner
    • - type
    • - length
    • - location of data
  • position (fd) and length are limited by C types used by kernel, but filesystem implemention can override that
  • files can be opened "simultaniously" by the same or other process just reffering to the file descriptor which is unique inside the system
  • no restrictions on concurrent file access, applications must handle syncronization
  • directories maps inodes to filenames (it is the name how Linux system finds the files for a user)
  • name + inode = link (stored in table/hash managed by kernel)
  • conceptually the directory is viewed like any normal file, but it contains only a mapping of names and inodes
  • from the filename the kernel finds inode
  • absolute pathname or relative pathname + working directory
  • multiple names can resolve to the same inode (hard link)
  • symbolic links more overhead, because first resolve file, so the linked-to file
  • symbolic links require specific system calls
  • Special files: block device files, character device files, named pipes and Unix domain sockets (Linux provides a system call to create a special file)
  • device files are block and character
  • named (FIFO special file) and regular pipes (in memory with system call)
  • sockets (Unix domain sockets, internet sockets)
  • the single, shared namespace for all mounted filesystems (partitions)
